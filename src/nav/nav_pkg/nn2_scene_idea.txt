Нейросеть №2: Топологическая карта (Семантическое понимание)
​Суть: Ваша база данных контекста. Векторная ассоциативная память.
​Что она ищет: Не конкретные объекты, а «суть» всей сцены целиком (комбинацию лесополосы, поля, горизонта и текстур).
​Тип модели: Базовая Foundation-модель (DINOv2) + агрегатор признаков (AnyLoc) + векторная база данных (FAISS).
​Принцип работы (Тактическая навигация):
​Модель сжимает весь кадр в один емкий массив чисел (глобальный дескриптор).
​Сравнивает его через FAISS с базой данных дескрипторов, собранных при разведывательном облете.
​
Как построить эту карту (Обучение "с учителем")


Открыть записанный ROS bag ( в репе уже есть код записи в rosbag)
​Пойти по таймлайну и, скажем, 2 раза в секунду будет выхватывать кадр с камеры.
​В эту же миллисекунду (используя message_filters для синхронизации времени) выхватит текущую позу из топика VINS/GPS и текущий кватернион из топика IMU.
​Прогонит кадр через DINOv2 -> запишет вектор в FAISS, а цифры (данные imu, gps координаты) в .json (пока .json).


Архитектура нашей базы знаний (FAISS + Metadata)
​FAISS спроектирован так, что он хранит только векторы и их порядковые номера (ID). Он не умеет хранить текстовые или числовые метаданные. Поэтому наша база данных будет состоять из двух синхронизированных файлов (или таблиц):
​Индекс FAISS (map.index): Массив векторов для молниеносного косинусного поиска. Выдает ID наиболее похожего места.
​База метаданных (пока metadata.json): Словарь, где по этому ID лежит физический контекст.
​Структура строки в базе метаданных:
ID -> [Timestamp, X (Lon), Y (Lat), Z (Alt), Qx, Qy, Qz, Qw (Кватернион IMU)]
