NN2 — топологическая карта (семантическая локализация). Как это устроено.
================================================================================

ИДЕЯ
DINOv2 сжимает весь кадр в ОДИН вектор (глобальный дескриптор «сути» сцены:
лесополоса+поле+горизонт+текстуры). FAISS ищет ближайший такой вектор в карте,
собранной на разведывательном облёте. По найденному месту знаем его физический
контекст: позу (X,Y,Z) и кватернион ориентации. Сеть обучать НЕ нужно (DINOv2
предобучен self-supervised).

ЦЕЛЬ (итог): ВОССТАНОВЛЕНИЕ VINS-ТРЕКИНГА ПОСЛЕ ПОТЕРИ. Когда VINS срывается/
ресетится, текущий кадр -> ближайшее место карты -> его поза -> переинициали-
зация абсолютной позиции. (NN1 = частая засечка по точечным ориентирам; NN2 =
грубая релокализация «по всей сцене», когда VINS вообще потерялся.)


СТАТУС (Инкремент 1)
ГОТОВО: пайплайн кадр -> DINOv2 -> FAISS -> поза места; каждую итерацию поза
   уходит в ноду relocalizer (пока ПУСТАЯ заглушка). Допущение: VINS на треке,
   мы просто отдаём найденную позу (кватернион + IMU + GPS/ENU координаты).
   Параллельно метка места идёт в /nn2/scene (баннер оператору, openhd_streamer).
ОСТАЛОСЬ:
   - ПРОГНАТЬ обучение топографа на реальных bag'ах (код головы и тренера
     ГОТОВ, см. ниже) + подобрать гиперпараметры/проверить eval_isometry;
   - детектор потери трекинга VINS (когда вообще звать релокализацию);
   - применение позы: relocalizer -> поправка в ray_tracer (единственный мост
     VINS->FCU, чтобы не было двух источников vision_pose);
   - общий FAISS-префильтр для NN1 (см. nn1_anchor_howto.txt, КРИТИЧНЫЕ МЕСТА).


МЕТРИЧЕСКИЙ MLP-«ТОПОГРАФ» (ядро точности; ГОЛОВА РЕАЛИЗОВАНА, ждёт обучения)
Проблема голого DINOv2: эмбеддинг гладкий, но НЕ метрический — L2-расстояние
между векторами не пропорционально метрам (тень на дороге может «раздвинуть»
кадры сильнее, чем реальный пролёт). Локально корреляция с перемещением есть,
глобально ломается.
Решение: маленький MLP поверх DINOv2, обученный ОФЛАЙН на ПК контрастивно
(Triplet Margin Loss + дистанц-регрессия) на дельтах одометрии VINS из bag'а:
   anchor = кадр t; positive = кадр t+0.5с (близкий по VINS);
   negative = кадр t+10с (далёкий). Штраф, когда L2 в латентном пространстве
   не совпадает с физической дельтой из VINS.
Итог — (локальная) изометрия: 1 метр пролёта ~ фиксированный сдвиг вектора.
Глобально ограничено алиасингом/ракурсом — нужна верификация NN1 (см. идею).

ЧТО УЖЕ В КОДЕ (scene_descriptor.py):
  - MetricHead(in_dim,hidden,out_dim) — сам MLP (LayerNorm+GELU), save/load
    самоописательных весов .pt (config+state_dict). Тренер их импортирует.
  - SceneEncoder(mlp_path=...) — есть веса -> encode() прогоняет CLS через голову,
    выход метрический (encoder.metric='l2', dim=out_dim, нормировка отключена);
    нет -> прежний сырой DINOv2 (metric='ip'). Голова общая для ноды и сборщика
    карты, поэтому карта и онлайн = одна связка автоматически.
  - SceneMatcher — путь к весам берёт из metadata.mlp; query() для L2-карты меряет
    дистанцию В МЕТРАХ и режет по max_dist (для IP — косинус по min_score).
  - build_scene_map.py --mlp — пишет IndexFlatL2, кладёт веса рядом с картой,
    проставляет metadata.mlp/metric. eval_isometry.py --mlp — замер DINOv2+MLP.
ТРЕНЕР — tools/train_topograph.py (ГОТОВ):
  Бэкбон DINOv2 заморожен, учим только MetricHead. Один раз считает CLS-фичи
  кадров + позы VINS (опц. кэш .npz), затем шаги SGD. Лосс = дистанц-регрессия
  (smooth_l1: ||Δhead|| -> метры; задаёт масштаб) + Triplet Margin (порядок).
  Пары/тройки — по ВРЕМЕННЫМ офсетам внутри одного пролёта: positive≈t+pos_dt,
  negative≈t+neg_dt, регрессия — случайный офсет [1..k_neg]. Кросс-bag пар нет
  (позы пролётов не co-registered).
    python3 train_topograph.py --bag ~/f1 ~/f2 --out topograph.pt --steps 4000
    python3 build_scene_map.py --bag ~/flight_bag --mlp topograph.pt
    python3 eval_isometry.py   --bag ~/flight_bag --mlp topograph.pt   # замер


КАРТА (data/scene_map/) — ДВА синхронизированных файла
FAISS хранит только векторы + порядковые ID, метаданные не умеет. Поэтому:
  map.index      — FAISS IndexFlatIP по L2-нормированным векторам (= косинус).
                   Строка i индекса <-> entries[i]. Бинарный, НЕ коммитится.
  metadata.json  — {model, dim, metric, origin, entries:[...]}, где entry =
                   {id, stamp, x, y, z, qx, qy, qz, qw, label}. Коммитим (схема
                   + сэмпл). origin = датум VINS<->гео (как у NN1).
ВАЖНО: размерность векторов и модель в metadata должны совпадать с моделью ноды
(иначе SceneMatcher отбракует карту). dim(dinov2_vits14)=384.


КАК СОБРАТЬ КАРТУ ИЗ BAG (обучение «с учителем»)
  tools/build_scene_map.py — идёт по таймлайну bag'а, ~2 раза/с выхватывает
  кадр, ассоциирует с БЛИЖАЙШЕЙ по времени позой VINS (одометрия) и кватернионом
  IMU, гонит кадр через DINOv2 -> вектор в map.index, контекст -> metadata.json.
    python3 build_scene_map.py --bag ~/flight_bag --rate 2.0 \
        --image-topic /image_color \
        --odom-topic /vins_estimator/odometry \
        --imu-topic /mavros/imu/data
  (bag пишет уже существующий auto-bag, см. distro/.) message_filters синхронит
  ЖИВЫЕ подписки; в офлайн-bag берём проще — последнее виденное odom/imu на
  момент кадра (воспроизводимо).

  tools/generate_mock_map.py — мок-карта (СЛУЧАЙНЫЕ векторы) для проверки
  ОБВЯЗКИ (nn2_scene -> relocalizer), путей и форм. Реальной локализации не даёт.


ОНЛАЙН (нода nn2_scene)
  Таймер ~3 с (NN2 редкая, fps видео не гейтит), держит последний /image_color.
  SceneMatcher: DINOv2(кадр) -> FAISS top-1 -> если косинус >= min_score, это
  место. Публикует:
    /nn2/scene          (String) — метка места, баннер openhd_streamer;
    /nn2/relocalization (String, JSON) — поза места -> relocalizer.
  JSON-в-String — заглушка («пока .json»); типизированный msg потребует
  отдельного CMake-интерфейс-пакета (nav_pkg — ament_python) — отложено.


ФАЙЛЫ
  nav_pkg/nn2_scene.py        — ROS-нода NN2 (Инкремент 1)
  nav_pkg/scene_descriptor.py — ядро: SceneEncoder (DINOv2 +опц.MLP) + MetricHead
                                + SceneMatcher (FAISS)
  nav_pkg/relocalizer.py      — ПУСТАЯ нода-заглушка (восстановление VINS)
  tools/build_scene_map.py    — БОЕВАЯ сборка карты из bag (DINOv2[+MLP])
  tools/generate_mock_map.py  — МОК карты (случайные векторы) для обвязки
  tools/train_topograph.py    — обучение MLP-«топографа» (MetricHead) из bag
  tools/eval_isometry.py      — замер изометрии эмбеддинга (до/после MLP)
  tools/visualize_fiber.py    — визуализация позиционного fiber из bag (PCA)
  tools/nn2_topograph_theory.txt — геометрия: fiber/нуисансы, почему работает
  data/scene_map/metadata.json— схема + сэмпл (map.index/.pt генерятся, не коммитятся)


КРИТИЧНЫЕ МЕСТА (Jetson Orin)
  .to(device) + torch.no_grad(): тензоры на GPU, без графа градиентов — иначе
      инференс секунды и риск OOM (общая боль с NN1).
  torch.hub: первый запуск DINOv2 тянет код+веса из интернета в ~/.cache/torch.
      На борту без сети — прогреть кэш заранее (или вендорить веса).
  Одна модель на карту и онлайн: вектор карты и вектор запроса обязаны считаться
      ОДНОЙ сетью (SceneEncoder общий для ноды и build_scene_map.py).
