================================================================================
NN2 — как ИЗМЕРИТЬ изометрию эмбеддинга на bag'ах (число вместо веры).
================================================================================

ЗАЧЕМ
Прежде чем верить, что «латентное расстояние = метры», это надо ПОСЧИТАТЬ.
Оценка даёт: (а) насколько пространство метрично глобально/локально, (б) где
оно «растянуто неравномерно», (в) зоны перцептивного алиасинга (главная угроза
релокализации), (г) сквозную ошибку локализации в метрах. Гоняется офлайн на ПК
по тем же bag'ам облёта; сравнивает голый DINOv2 vs DINOv2+MLP-«топограф».

ВХОД
  - bag(и) облёта: image-топик + одометрия VINS (поза) + IMU (yaw).
  - функция эмбеддинга f(): SceneEncoder (DINOv2, потом DINOv2+MLP).
  ⚠ Несколько bag'ов: позы РАЗНЫХ пролётов сперва свести в ОДИН кадр
    (pose-graph с межпролётными loop closure) — иначе d_phys между пролётами
    не определено. Один bag — кадр VINS внутренне согласован (только локальный
    дрейф), межпролётную часть метрик пропускаем.

ШАГ 1. ВЫБОРКА
  ~2 Гц с таймлайна (как build_scene_map.py). Для каждого кадра i:
    f_i  = encode(кадр)            # вектор
    p_i  = (x,y,z) из VINS odom    # метры
    yaw_i из IMU                    # для разбивки по ракурсу

ШАГ 2. ПАРЫ
  Берём пары (i,j) (все или подвыборка — O(N^2) режем рандомом). Для каждой:
    d_lat  = || f_i - f_j ||        # L2 в латенте (для изометрии именно L2)
    d_phys = || p_i - p_j ||        # метры
    d_yaw  = угловая разница курсов
  Окно: считаем метрики на d_phys в [0..D] (напр. D=50 м) — там VINS-дельты
  надёжны и корреляция вообще ожидаема. Дальние пары — отдельно (см. дрейф).

ШАГ 3. МЕТРИКИ ИЗОМЕТРИИ
  (1) Scatter d_lat vs d_phys (PNG) — глазами: линия или каша.
  (2) Spearman ρ  — МОНОТОННОСТЬ (хватает для ретривала «ближайший = ближний»).
      Pearson r   — ЛИНЕЙНОСТЬ (ближе к изометрии).
      Высокий ρ + низкий r = годно для NN-поиска, но НЕ метрично.
  (3) Линейная подгонка d_lat = α·d_phys + b.
      α = «единиц вектора на метр». Остаточный разброс σ_res; переведи в метры:
      σ_res/α = СКО ошибки в метрах. Изометрия -> тугая линия, b≈0.
  (4) Стабильность масштаба α ПО КАРТЕ: фитим α локально (по регионам/сегментам),
      смотрим разброс α. Глобальная изометрия <=> α ~ const везде. Большой
      разброс α = «неравномерно растянуто» (та самая глина) -> сюда MLP.
  (5) Kruskal stress-1 = sqrt( Σ(d_lat - α·d_phys)^2 / Σ d_lat^2 ).
      Одно число, 0 = идеальная изометрия. До/после MLP — главный показатель.

ШАГ 4. АЛИАСИНГ (самый важный диагноз)
  Ищем пары: d_lat МАЛОЕ, но d_phys БОЛЬШОЕ (выглядят одинаково — разные места).
    критерий: d_lat < ε (радиус NN) И d_phys > D_alias (напр. 30 м).
  Доля таких пар + ГДЕ они на карте = зоны, где релокализация защёлкнется не туда.
  Их MLP не лечит (вход одинаков) -> туда геометрическая верификация NN1.

ШАГ 5. СКВОЗНАЯ ОШИБКА ЛОКАЛИЗАЦИИ (что реально нужно)
  Эмулируем релокализацию: query-кадр -> NN-поиск в карте -> top-1 -> ошибка =
  || p_query - p_nn || в метрах. По всем query:
    median, 90-й перцентиль, recall@Xм (доля query с ошибкой < 5/10/20 м).
  Это конечное число важнее абстрактной изометрии. Считать для DINOv2 и
  DINOv2+MLP — увидишь прирост от выпрямителя.
  ⚠ Честно: query и map из РАЗНЫХ пролётов (разный свет/время). Leave-one-out
  внутри одного bag завышает результат.

ШАГ 6. ПО РАКУРСУ
  Повтори (5), сгруппировав по |d_yaw|. Если ошибка резко растёт при большом
  d_yaw — позиционной изометрии мешает курс; вывод: условие на yaw или карта
  по ячейкам (позиция × курс).

ЧТО СЧИТАТЬ УСПЕХОМ (ориентиры)
  - Spearman ρ > 0.9 локально — ретривал健ный.
  - stress-1 заметно падает после MLP (напр. 0.4 -> <0.15).
  - разброс α по карте схлопывается после MLP.
  - recall@10м растёт; медиана ошибки в пределах шага кадров карты.
  - доля алиас-пар мала И локализована (передать в NN1).
