NN1 — якорная локализация (замена GPS). Как это устроено.

ИДЕЯ
SuperPoint+LightGlue сопоставляет текущий кадр дрона с базой кадров облёта, у
которых известны реальные координаты. Нашли известный ориентир -> ray tracing
считает абсолютную позицию дрона -> сравниваем с тем, что насчитал VINS ->
разница = дрейф -> сброс. Нейросеть обучать НЕ нужно (SuperPoint/LightGlue
предобучены).


СОПРЯЖЁННАЯ СИСТЕМА КООРДИНАТ (ключевое)
Есть две рамки, сшитые в точке взлёта:
  - рамка VINS  — метрическая, начало = точка взлёта = (0,0,0);
  - гео-рамка   — реальные GPS (lat/lon) + высота по карте высот.
Точка взлёта = ОРИЕНТИР №0: держит ОБЕ координаты одновременно —
  (0,0,0)_VINS  <->  (lat0, lon0, alt0)_гео.
Это и есть «сопряжение»: зная пару на взлёте, переводим между рамками.

Ориентиры в базе хранят реальные GPS+высоту (гео-рамка). Когда SuperPoint
находит ориентир:
  1. ray tracing по известным GPS ориентира + углам камеры (крен/тангаж из
     полётника) + высоте (баро) -> абсолютная позиция камеры в гео-рамке;
  2. через датум взлёта переводим в рамку VINS;
  3. сравниваем «что думает VINS» (его счисленный путь) с «что насчитал ray
     tracing» -> дрейф -> коррекция.


БАЗА (data/reference_db/)
database.json:
  origin      — точка взлёта = ориентир №0 (GPS+высота, датум VINS<->гео)
  landmarks   — id -> {lat, lon, alt}            (реальные гео-координаты)
  references  — имя_файла.npz -> id_ориентира
*.npz (выхлоп SuperPoint по кадру облёта):
  keypoints (N,2) float, descriptors (N,256) float, image_size (2,) float

Как наполняется: делаем кадры (ROI) ориентира с разных сторон -> прогоняем
через SuperPoint -> по файлу на кадр. Координаты ориентира проставляем руками
(облёт с известной геопривязкой).


ПАЙПЛАЙН
Онлайн (нода nn1_anchor):
  SuperPoint на /image_color -> LightGlue матчит против эталонов базы. Если
  совпало > min_matches точек -> по references узнаём id, по landmarks — гео.
  Из пиксельных keypoints совпадений строим bbox -> публикуем в
  /nn1/detections (vision_msgs/Detection2DArray, class_id = id ориентира).
  Bbox сразу видно в оверлее openhd_streamer.


ИНКРЕМЕНТЫ
  [1] (этот) детектор: SuperPoint+LightGlue -> Detection2DArray (bbox + id).
  [2] ray-tracing нода: bbox+id -> координаты по id -> луч (intrinsics + углы +
      баро) -> абсолютная позиция -> сравнение с VINS -> сброс дрейфа.

КРИТИЧНЫЕ МЕСТА (Jetson Orin)
  .cuda()/.to(device): тензоры обязательно на GPU — иначе инференс секунды,
      а не миллисекунды.
  torch.no_grad(): без него PyTorch строит граф градиентов -> 6 ГБ памяти
      Orin переполняются -> OOM.
  Масштаб базы: брутфорс LightGlue по всем эталонам не тянет тысячи кадров.
      Нужен ретривал-префильтр (DINOv2/AnyLoc + FAISS, общий с Нейросетью №2):
      выбирает top-K кандидатов, LightGlue — только по ним. Сейчас брутфорс
      (TODO в anchor_matcher.py) — ок только для крошечной базы.
