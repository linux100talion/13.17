# FLOW-DAMP — спецификация (черновик v0)

Scale-free velocity-демпфер от ветра по ОДНОЙ форвард-камере. Переключаемый режим:
включается у трудного места, игнорирует VINS/одометрию, гасит снос, пропускает стики.
Концептуальное обоснование целиком — `FAQ_vins.md` (разделы 6–15). Связано: `todo4.txt`
(срыв VINS при малом движении), `src/lab/CLAUDE.md` (gz-hold как референс-костыль).

Легенда: ✅ — твёрдое (выводится из разобранного), ⚠️ — открыто (нужно решение).

---

## 1. Назначение ✅

Переключаемый режим: включается у трудного места. Scale-free velocity-демпфер по одной
форвард-камере, ИГНОРИРУЕТ одометрию/VINS, РЕАГИРУЕТ на стики. Цель — гасить снос
(особенно от ветра) там, где VINS хрупок/срывается (малое движение). Комплементарен VINS,
не замена: выживает ровно в том режиме, где VINS дивергирует (FAQ_vins п.12 Q1).

## 2. Границы — чего НЕ делает ✅

- **не** position hold — только velocity damping → медленный дрейф позиции остаётся;
- **не** управляет наклоном — pitch/roll держит FCU (rate=гироскоп, level=гироскоп+
  акселерометр через AHRS); камера для углов не нужна (FAQ_vins, наблюдаемость);
- **не** метрический, **не** через EKF3 — это путь (B) из FAQ_vins п.15: прямой регулятор
  поверх ALT_HOLD (scale-free flow в EKF3 не лезет — он хочет метры).

## 3. Тракт ✅

```
базовый режим: ALT_HOLD (баро-высота, self-level от FCU)
нода = расширение alt_hold_bootstrap.py (флаг --flow-hold), реюз RC override 20 Гц
       (каналы 1-2 = roll/pitch)
вход:  /image_mono (или /image_color) + gz-IMU гироскоп (FLU) + extrinsicRotation
пайплайн:
  optical flow → derotate (ω_cam = R · ω_imu) → остаточный ТРАНСЛЯЦИОННЫЙ поток
    → управляющие сигналы → PID → PWM-offset roll/pitch → RC override
```

## 4. Сигналы → оси (решено — фазировка)

Философия: **v0 = только сильные, высоко-SNR оси; слабые (looming, визуальный yaw) — фаза
2 поверх уже работающего ядра.** Так легче писать и отлаживать (изоляция переменной).

| Ось | v0 | фаза 2 |
|---|---|---|
| **боковой снос** | ✅ ДЕМПФЕР (главное): средний горизонт. поток → ROLL | — |
| **продольный** | пилот (сквозной стик) | looming-зануление: дивергенция → PITCH, гейт по confidence |
| **yaw** | пилот (сквозной), derotation защищает боковой сигнал | визуальное удержание курса |
| **поток** | sparse LK + аффинный фит поля | — |

Обоснование решений (детали — обсуждение + FAQ_vins):
- **боковой → ROLL** — крепкая, высоко-SNR ось; ядро v0;
- **продольный → пилоту в v0**: looming — наихудшей обусловленности (FAQ_vins п.9: ноль в
  центре, низкий SNR, вырождается в дальнем поле) + риск тряски pitch (pitch двигает обзор
  → обратная связь). looming-алгоритм в фазе 2 ТОТ ЖЕ, но ложится на доказанное ядро +
  переиспускает готовый confidence-гейт. Грубую низко-авторитетную версию в v0 НЕ делаем
  (шумный pitch-контур уходит в обратную связь — лучше ноль, чем грубый looming);
- **yaw → пилоту в v0**: derotation по гироскопу УЖЕ снимает yaw-вращательный поток из
  бокового сигнала → визуальный yaw-hold для работы демпфера НЕ нужен (это отдельная
  плюшка, FAQ_vins п.7, требует развязки bas-relief п.8). При активном yaw треки рвутся →
  confidence падает → §7-гейтинг приглушает (это и нужно).

Целевой сценарий v0 — **A (пролёт между препятствиями)**: боковой снос критичен (увод на
кусты), вперёд рулит пилот. Сценарий B (подход к стене, продольный критичен) — это и есть
фаза 2 (looming становится основным контуром).

## 5. IMU-вход ✅

- **только гироскоп** (gz-IMU, FLU, `/gz_imu/data_flu`), **только для derotation**;
  акселерометр НЕ нужен (мы scale-free, attitude не оцениваем);
- `extrinsicRotation` из `sim.yaml` (R камера↔IMU) — перевод ω в фрейм камеры; нужна
  только ротация (рычаг на ω не влияет — твёрдое тело, одна ω везде);
- `extrinsicTranslation` [0.15,0,0.05] — ОПЦ. компенсация рычага `v_рычаг = ω×r` (камера
  смещена от IMU → при вращении едет по дуге → фейковая трансляция); или гейтинг по ω;
- фрейм-цепочка: `gz-IMU (FRD) → imu_frd_to_flu.py → FLU → extrinsicRotation → optical`,
  та же `extrinsicRotation`, что у VINS (согласованность с триангуляцией).

## 6. Стик = setpoint ✅

- стик задаёт **желаемый поток** (scale-free); `error = измеренный − заданный`, PID → error→0;
- центр стика → desired=0 → зануляем всё (держим, гасим ветер); отклонён → desired=X → летим;
- демпфер борется ТОЛЬКО с ветром (некомандованной скоростью), команды проходят
  (FAQ_vins п.12 Q2 — стиль LOITER/FLOWHOLD, но в flow-единицах);
- бонус: у близких препятствий тот же стик = меньшая реальная скорость (само-торможение).

## 7. Безопасность ✅

- **детектор качества потока** (мало фич / темно / aperture) → **плавный fade-out в чистый
  ALT_HOLD** (ручной), а не лить мусор в override (аналог failureDetection VINS, но
  доброкачественный — нода простаивает, не дивергирует);
- **гейтинг при большом ω** (активный поворот), чтобы рычаг и ошибка derotation не дёргали
  демпфер (реюз паттерна из alt_hold_bootstrap.py: yaw не льётся одновременно с translate).

## 8. Реюз существующего кода ✅

- `src/lab/alt_hold_bootstrap.py`: тракт RC override (20 Гц, `_wall_publish`/`tick`),
  ALT_HOLD, arm/climb/land, PID-блок `--gz-hold` (~277–330 — меняется ТОЛЬКО источник
  скорости: поток вместо gz-истины), паттерн yaw-гейтинга (~351–356);
- `src/vins/.../config/sim.yaml`: extrinsic (R + t);
- `/gz_imu/data_flu` + `imu_frd_to_flu.py`: FLU-гироскоп;
- `src/lab/capture_scene.sh`: атомарный прогон для приёмки.

## 9. Приёмка / тест-план ✅

- **оракул истины** — истинная поза Gazebo (`/model/iris_cam/odometry`): сравнить дрейф
  позиции с демпфером и без него;
- **ветер**: `SIM_WIND_*` → измерить, насколько демпфер срезает скорость сноса;
- **проброс стиков**: отклонить → летит в нужную сторону; центр → гасит;
- **fade-out**: убрать текстуру/затемнить → плавный возврат в ALT_HOLD без рывка.

## Решено (1–3, см. §4)

1. **Источник потока**: ✅ **sparse LK + аффинный фит** (агрегаты: средний поток +
   дивергенция + curl; реалтайм на 10 Гц; число треков = confidence; свой LK, независимый
   от VINS — переживает ребут VINS). dense — перебор ради агрегатов, тяжело на CPU-боксе.
2. **Продольная ось**: ✅ **пилоту в v0**, looming-зануление — фаза 2 (тот же алгоритм
   поверх доказанного ядра + готовый confidence-гейт).
3. **Yaw**: ✅ **сквозной стик в v0** (derotation защищает боковой сигнал), визуальный
   yaw-hold — фаза 2.

## Решено (4–5)

4. **Топик/частота камеры**: ✅ **`/image_mono`** (8-бит grayscale, вход VINS) — LK всё
   равно работает по grayscale, тот же источник что VINS, минимум оверхеда. Частота —
   камер-rate (~10 Гц в sim, Найквист 5 Гц лимитирует полосу демпфера; на боевой быстрее).
   Поток считается между соседними кадрами; derotation интегрирует гироскоп 250 Гц за
   интервал кадра.
5. **Инжект**: ✅ **RC override** (`/mavros/rc/override`, каналы 1-2 roll/pitch), реюз
   тракта alt_hold_bootstrap.py. Альтернативы отброшены: `VISION_SPEED_ESTIMATE`→EKF3 =
   путь (A), требует метрики (мы scale-free); `SET_ATTITUDE_TARGET` обходит ALT_HOLD и его
   логику стиков. RC override поверх ALT_HOLD — проверенный в стеке путь.
