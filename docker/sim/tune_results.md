# FLOW-DAMP — тюнинг гейнов (гибридный харнес)

Журнал прогонов тюнинга бокового флоу-демпфера. Харнес и теория —
`HowToFlow_PID_synth.md` (синтетика A+B+C) и `FAQ_pid.md` (почему ki=0).

Харнес: `BS_GZHOLD=1 BS_FLOWHOLD=1` — gz держит продольную (pitch) истиной Gazebo,
флоу демпфирует боковую (roll). Метрика — абсолютная боковая RMS_v по drift_check.py
(SAFE_SEC=30, R_SAFE=40). Продольная gz-held ≈ 0.1 (норма — нормировка не применима).

Зафиксировано: `ki=0` ✅ (ki>0 = пружина+windup → растущая осцилляция),
`osign=+1` ✅ (osign=-1 = положительная ОС → разгон-убегание).

## Вывод после O1
- `osign=+1` — правильный демпфирующий знак (O1 доказал: −1 = разгон до 33 м/с).
- Знак НЕ причина дисперсии H2(0.41)↔H4(2.40) при идентичном конфиге.
- (ГИПОТЕЗА, позже опровергнута синтетикой) лаг-индуцированная маргинальная устойчивость,
  направление «kp вниз». O2 её опроверг.

## Вывод после O2 (пересмотр)
- «kp вниз» ОПРОВЕРГНУТ: kp4 слишком слаб (снос 42 м, v до 6.35 м/с, roll_off ≤26).
- O2 — НЕ осцилляция, а один медленный сброс: e_right монотонно 0→42.8, потом v
  разворачивается (−1.85) → osign=+1 = чистая отриц. ОС, просто слабая/медленная.
- Центральная загадка H2(0.41) vs H4(2.40): один конфиг, два исхода. Боковой замкнутый
  контур недетерминирован (тайминг обработки кадров flow — wall-clock), sim-override
  детерминизировал только продольную → одиночный прогон ненадёжен.

## СИНТЕТИКА A+B+C (коммиты 442d07d, 3fbc2f3) — разрыв тупика
Полный прогон дорог (RTF 0.07, ~5 мин) и недетерминирован → тюнить им нельзя.
Три детерминированных оффлайн-теста (см. `HowToFlow_PID_synth.md`):
- **A** (`flow_synth_test.py`): rsign=+1 и матрица R подтверждены детерминированно
  (без Gazebo) — синтетические пары кадров с известным сдвигом/поворотом.
- **B** (`flow_loop_sim.py`): numpy-ОДУ контура + ТОТ ЖЕ PID. osign−1→runaway (=O1);
  P-only ограничивает v (`v_term=d/(|k|·kp·s)`), выше kp → ниже v_term. Осцилляции
  P+лаг нет (пружина слишком медленная для реалистичного τ) → уносы O2/H4 = слабый kp,
  не P-нестабильность.
- **C** (`flow_calib.py`): system-ID планта из O2-бэга, ВАЛИДНА (пред 4.2 vs набл
  2.76 м/с). Параметры: s=0.807 px/(м/с), bias=−0.5px, noise=1.65px, k=−0.0196,
  d=0.266, τ≈0.
  ⚠ **R²(flow↔v)=0.19** — ПОТОК СЛАБЫЙ ПРОКСИ СКОРОСТИ (81% шум). Это потолок тюнинга.

## Направление (пересмотр по калиброванному B) — kp ВВЕРХ
Калиброванный свип (ki0 osign+1): kp2=5.53(РАСХ) kp8=1.44 kp16=0.47 kp24=0.19 kp32=0.18.
→ Целевой режим: **kp16-24, ki0, osign+1** → RMS_v ~0.2-0.5 м/с, дрейф ограничен.
(H3 kp16 «хуже» был одиночным недетерминированным прогоном — калибровка его пересматривает.)

## Следующий эксперимент
- ПОДТВЕРДИТЬ предсказание ОДНИМ реальным гибридом kp16 ki0 osign+1. Ждём RMS_v~0.4-0.5.
  Если совпадёт → калибровка надёжна, гейн найден. Параллельно — атаковать R²=0.19
  (качество потока: медиана по кадрам / RANSAC вместо raw-median / отбор фич).
