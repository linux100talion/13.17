4. ДВИЖЕНИЕ для параллакса — вести СЕТПОЙНТ, не «пробивать» PID (КЛЮЧЕВОЕ)
------------------------------------------------------------------------
Вопрос «как пошевелить прибитый дрон»: команды НЕ прорываются через PID — он их
давит (это и есть стабильность; раскачка «поверх» петли гасится). Правильно —
двигать САМ СЕТПОЙНТ:
  setpoint(t) = центр + траектория(t)   # напр. круг R, период T
PID отслеживает движущуюся цель → дрон едет по пути, оставаясь ОГРАНИЧЕННЫМ (петля
держит его НА траектории, не даёт улететь). Это замена прежнего open-loop excite
(он убегал, т.к. разомкнут) на ЗАМКНУТОЕ слежение — то же движение, но безопасное.

VINS нужен ПАРАЛЛАКС = трансляция (чистый поворот не даёт масштаб) + вариация
ускорения. Значит водить сетпойнт малым горизонтальным узором:
  - круг R≈1–3 м, период несколько сим-сек; или восьмёрка / sweep.

  [ ] добавить в gz-hold режим траектории: hold_sp = центр + круг(R,T),
      параметры BS_GZ_TRAJ_R / BS_GZ_TRAJ_T;
  [ ] включить его для VINS-init (движение + 250Гц IMU вместе).

Аналогия с боевым бортом: там это GUIDED + setpoint_position/local, когда позицию
даст реальный VINS. gz-hold + траекторный сетпойнт — sim-прокладка под ту же логику.
