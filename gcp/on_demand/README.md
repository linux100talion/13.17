# gcp/on_demand — on-demand T4 по зонам (тонкие start/stop/ssh)

Скрипты питания **on-demand** (НЕ вытесняемого) GPU-инстанса `dev-workspace-1317`
(Tesla T4) по зонам региона `europe-west4`. on-demand дороже SPOT, но не
вытесняется — инстанс живёт постоянно, поэтому скрипты **тонкие**: никакой
снапшот-логики и переезда между зонами (это забота `../spot/`).

## Скрипты

По каждой зоне (`a`, `b`, `c`) — тройка, отличающаяся внутри зоны только
`TARGET_ZONE`:

| Скрипт | Что делает |
|---|---|
| `{a,b,c}_start.sh` | `gcloud instances start` инстанса в своей зоне (только старт существующего) |
| `{a,b,c}_stop.sh`  | погасить инстанс в своей зоне (диск остаётся, платишь за storage) |
| `{a,b,c}_ssh.sh`   | SSH в инстанс своей зоны (доп. аргументы → `gcloud compute ssh`) |

Соседние верхнеуровневые скрипты зовутся через `../` (`../01_create_workspace.sh`,
`../02_power_manager.sh`, `../04_check_money_leak.sh`).

## Чем отличается от `../spot/`

| | `on_demand/` | `../spot/` |
|---|---|---|
| Provisioning | on-demand (не вытесняется) | SPOT (вытесняемый, дешевле) |
| `start` | тонкий: только `instances start` | «комбайн»: снапшот → перенос диска по зонам → create |
| Диск между зонами | **не трогает** (переезд — в spot/) | переносит снапшотом |
| Создание инстанса | отдельно: `../01_…` / `../08_…` | создаёт сам при ловле |

**Границы тонких скриптов:** `start` поднимает только СУЩЕСТВУЮЩИЙ инстанс в
СВОЕЙ зоне. Если инстанса нет в этой зоне — скрипт подсказывает (создать через
`../01_create_workspace.sh` / `../08_add_gpu.sh` либо перенести зону через
`../spot/`), но сам ничего не переносит и не создаёт. Если on-demand-ёмкость T4
в зоне исчерпана (`RESOURCE_POOL_EXHAUSTED`) — повторить позже или ехать в другую
зону через `../spot/` (там вся работа с зональным диском/снапшотами).

## Типовые сценарии

```bash
./a_start.sh                      # поднять инстанс в europe-west4-a
./a_ssh.sh                        # подключиться
./a_ssh.sh --command='nvidia-smi' # одноразовая команда
./a_stop.sh                       # выключить на ночь (диск цел)
```

Перенос диска между зонами и менеджмент снапшотов — см. `../spot/README.md`.
