# gcp/snapshot — снять бэкап-снапшот boot-диска по зонам

Тонкие скрипты, снимающие **датированный бэкап-снапшот** boot-диска
`dev-workspace-1317` в своей зоне региона `europe-west4`.

## Скрипты

| Скрипт | Что делает |
|---|---|
| `{a,b,c}_take.sh`    | снять снапшот диска `dev-workspace-1317` в зоне a/b/c |
| `{a,b,c}_restore.sh` | восстановить диск `dev-workspace-1317` в зоне a/b/c из снапшота |

Внутри зоны скрипты отличаются только `TARGET_ZONE`.

## Имя и параметры снапшота

- **Имя:** `dev-workspace-1317-snap-YYYYMMDD` (напр.
  `dev-workspace-1317-snap-20260624`). Буквы зоны в имени НЕТ: инстанс/диск живёт
  в одной зоне за раз, снапшотим единственный существующий диск — коллизии имён
  нет, а сам снапшот глобальный (его зона роли не играет). Тег даты
  переопределяется: `TAG=manual ./a_take.sh`.
- **Долгоживущий бэкап**, существующие снапшоты не трогаются. Это НЕ транзитный
  `dev-workspace-1317-snap` (его пересоздают `../spot/*_start.sh` при переезде
  между зонами).
- **Storage location** — региональный (= регион зоны диска): глобально доступен
  для `disks create` в любой зоне, дешевле мультирегиона.
- **Инкрементальные:** первый снимок полный, дальше только дельта — снимать часто
  дёшево.
- Снапшот можно снять и на **работающем** инстансе.
- **Идемпотентно:** если снапшот с таким именем уже есть — скрипт выходит.

## Уборка — на тебе

Скрипты только создают снапшоты и ничего не удаляют — старые бэкапы копятся и
занимают место (плата за storage). Чистить вручную:

```bash
gcloud compute snapshots list --filter="sourceDisk:dev-workspace-1317" \
  --project=drone-13-17-workspace-2026
gcloud compute snapshots delete <name> --project=drone-13-17-workspace-2026
```

## Восстановление: `{a,b,c}_restore.sh`

Создаёт диск `dev-workspace-1317` в своей зоне из снапшота (снапшот глобальный →
зона-источник роли не играет).

- **Какой снапшот:** по умолчанию самый свежий снапшот этого диска (по времени
  создания). Конкретный — `SNAPSHOT=<name> ./a_restore.sh`.
- **Защита существующего диска** (молча не затирает):
  - привязан к инстансу → отказ; освободить:
    `ZONE=<zone> ../05_delete_instance.sh` (отвяжет, boot сохранит), затем
    `REPLACE=1 ./a_restore.sh`;
  - свободен (отвязан) → перезатереть только с `REPLACE=1` (delete + create);
  - в зоне диска нет → просто создаёт.
- После восстановления поднять инстанс на диске: `../spot/<z>_start.sh`
  (создаст инстанс в этой зоне).

## Связь с другими каталогами

- `../spot/` — переезд диска между зонами через ТРАНЗИТНЫЙ снапшот (своя логика,
  своё имя). Здесь — независимые бэкапы + восстановление из них.
- `../spot/<z>_start.sh` — поднять инстанс на восстановленном диске.
