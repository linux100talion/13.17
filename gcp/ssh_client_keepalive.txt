================================================================================
  SSH keepalive НА КЛИЕНТЕ — чтобы idle-сессия к GCE не рвалась
================================================================================

ПРОБЛЕМА
  SSH-сессия к боксу отваливается при простое → вместе с ней умирает интерактив
  (Claude Code, tail -f, Monitor и т.п.). Фоновые задачи под `nohup`/tmux при этом
  ВЫЖИВАЮТ — рвётся только терминал.

ДИАГНОЗ (проверено на dev-workspace-1317, 2026-07-02)
  Сервер (sshd) НЕ виноват — он уже настроен держать idle:
      /etc/ssh/sshd_config.d/00-keepalive.conf:
          ClientAliveInterval 60
          ClientAliveCountMax 10080     # 60с × 10080 ≈ 7 суток простоя
          TCPKeepAlive yes
      (50-cloudimg-settings.conf даёт Interval 120, но sshd берёт ПЕРВОЕ значение —
       лексически 00-* раньше 50-* → в силе Interval 60 + CountMax 10080.)
  Значит обрыв идёт СО СТОРОНЫ КЛИЕНТА/СЕТИ: сон/крышка ноута, смена Wi-Fi,
  таймаут NAT/VPN/фаервола между клиентом и GCE. Серверные пробы держат NAT, но
  если клиент уснул или сменил сеть — TCP всё равно умирает. Лечится на клиенте.

--------------------------------------------------------------------------------
  ВАРИАНТ 1 — ~/.ssh/config на клиенте (для любого ssh, в т.ч. gcloud)
--------------------------------------------------------------------------------
  gcloud compute ssh под капотом дёргает системный ssh и читает ~/.ssh/config.
  Добавить (Host * или конкретный хост):

      Host *
          ServerAliveInterval 60
          ServerAliveCountMax 10080
          TCPKeepAlive yes

  Клиент сам шлёт пробы каждые 60с → переживает короткие простои и NAT-таймауты.

--------------------------------------------------------------------------------
  ВАРИАНТ 2 — через gcloud --ssh-flag (разово/в обёртках gcp/*_ssh.sh)
--------------------------------------------------------------------------------
  Скрипты gcp/{02_power_manager.sh ssh, spot/*_ssh.sh, on_demand/*_ssh.sh}
  пробрасывают доп. аргументы в `gcloud compute ssh`. Можно передать флаги ssh:

      ./gcp/02_power_manager.sh ssh \
          --ssh-flag='-o ServerAliveInterval=60' \
          --ssh-flag='-o ServerAliveCountMax=10080'

  (аналогично для spot/a_ssh.sh и т.д. — доп. аргументы идут в gcloud). Вариант 1
  предпочтительнее: настроил один раз, работает везде.

--------------------------------------------------------------------------------
  ВАРИАНТ 3 — tmux/screen на боксе (настоящий фикс для долгих прогонов)
--------------------------------------------------------------------------------
  Развязывает работу от SSH-сессии: работа живёт в tmux на СЕРВЕРЕ и переживает
  ЛЮБОЙ обрыв. Переподключился → `tmux attach`.

      # на боксе:
      tmux new -s work      # или: tmux attach -t work
      # ... запускаешь долгие штуки внутри ...
      # отсоединиться: Ctrl-b d  (сессия и процессы продолжают жить)

  Именно из-за такой развязки наши свипы под `nohup` (напр. src/lab/yaw_tune_sweep.sh)
  переживают обрыв SSH — гибнет только Monitor в процессе Claude Code, привязанном
  к терминалу; сам прогон идёт дальше, метрики копятся в output/*.csv.

РЕКОМЕНДАЦИЯ
  Вариант 1 (постоянно) + Вариант 3 для долгих прогонов. Серверную сторону трогать
  не нужно — она уже настроена (00-keepalive.conf).
