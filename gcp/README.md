# GCP Dev Workspace — управление инстансом

Набор скриптов для создания и повседневного управления облачным рабочим инстансом
в Google Cloud Platform. Основная цель — удобная работа с GPU-машиной и экономия бюджета
(включаем только когда работаем, меняем мощность под задачу).

## Параметры инстанса

| Параметр    | Значение                      |
|-------------|-------------------------------|
| Project     | `drone-13-17-workspace-2026`  |
| Zone        | `us-central1-a`               |
| Instance    | `dev-workspace-1317`           |
| GPU         | 1× NVIDIA Tesla T4            |
| OS          | Ubuntu 24.04 LTS (amd64)      |
| Boot disk   | 120 GB, pd-balanced           |

## Требования

- Установленный и настроенный [`gcloud` CLI](https://cloud.google.com/sdk/docs/install)
- Выполненный вход: `gcloud auth login`
- Доступ к проекту `drone-13-17-workspace-2026`

## Скрипты

### `01_create_workspace.sh`
Первичное создание инстанса со всеми параметрами (GPU T4, n1-standard-8, диск 120 GB).
Запускается один раз при первоначальной настройке. Первый аргумент — размер
загрузочного диска в GB (по умолчанию 120).

```bash
./01_create_workspace.sh        # диск 120 GB (по умолчанию)
./01_create_workspace.sh 250    # диск 250 GB
```

> Инстанс создаётся с `--maintenance-policy=TERMINATE` — обязательно для машин с GPU.

### `02_power_manager.sh`
Повседневный менеджер питания. Запуск/остановка/подключение к серверу для экономии бюджета —
остановленный инстанс не тарифицируется за CPU/GPU (платишь только за хранение диска).

```bash
./02_power_manager.sh start   # запустить сервер
./02_power_manager.sh stop    # остановить сервер
./02_power_manager.sh ssh     # подключиться по SSH
```

### `03_resize_node.sh`
Изменение мощности инстанса «на лету». Останавливает инстанс, меняет тип машины и запускает снова.

```bash
./03_resize_node.sh light   # n1-standard-4 — правка кода, чтение логов
./03_resize_node.sh heavy   # n1-standard-8 — тяжёлая компиляция C++/ROS2
```

### `04_check_money_leak.sh`
Проверка статуса: не забыл ли я выключить сервер? Показывает, работает инстанс или остановлен.

```bash
./04_check_money_leak.sh
```

- `RUNNING` — ⚠️ сервер работает, биллинг идёт
- `TERMINATED` — 🛡️ сервер остановлен, деньги за CPU/GPU не списываются

### `05_delete_instance.sh`
Полное удаление инстанса с подтверждением. Загрузочный диск **сохраняется** (`--keep-disks=boot`).

```bash
./05_delete_instance.sh
```

> ⚠️ После удаления за хранение сохранённого диска биллинг продолжает идти.
> Запрашивает ввод `yes` перед удалением.

## Типичный рабочий день

```bash
./02_power_manager.sh start    # утром: включить сервер
./02_power_manager.sh ssh      # работать
# ... работа ...
./02_power_manager.sh stop     # вечером: выключить сервер
./04_check_money_leak.sh       # проверить, что точно выключен
```
